Spark RDD · Graph Algorithms

Social Network Community Detection with Girvan-Newman

Implemented the Girvan-Newman algorithm entirely in pure Spark RDD — no DataFrame/SQL, no graph libraries. BFS computes edge betweenness centrality across all source nodes; the algorithm iteratively removes the highest-betweenness edge and evaluates partition quality via the Modularity formula. Applied to a Yelp co-review social graph to surface communities with similar business taste patterns.

Apache Spark RDD Girvan-Newman BFS Betweenness Centrality Modularity

Python · BFR · Mahalanobis

BFR Clustering for High-Dimensional Large-Scale Data

Implemented the Bradley-Fayyad-Reina (BFR) algorithm for datasets too large to load into memory. Streams data in chunks while dynamically maintaining Discard Set, Compression Set, and Retained Set — each tracked by sufficient statistics (N, SUM, SUMSQ). Assigns new points via Mahalanobis distance against cluster distributions, and merges Compression Sets with Discard Sets in the final round.

Python NumPy BFR Algorithm Mahalanobis Distance Scikit-Learn

Python · Spark · Streaming

Real-Time Data Stream Mining Engine

Three core streaming algorithms implemented from scratch under strict memory constraints on an infinite simulated data stream. Bloom Filter: custom hash f(x)=(ax+b)%m with a bit array for membership testing. Flajolet-Martin: binary hash trailing-zero counting for distinct element cardinality estimation. Reservoir Sampling: dynamic fixed-size representative stream sample maintained with O(1) per element.

Python Apache Spark Bloom Filter Flajolet-Martin Reservoir Sampling

C · CUDA C · cuBLAS · Google Cloud

CUDA-Accelerated Computing & Python Shared Library

Hardware-level GPU optimization for large-scale matrix multiplication and 2D image convolution. Built a CPU baseline in C, then progressively ported to naïve CUDA and optimized with Shared Memory Tiling (TILE_WIDTH=16), achieving up to 1,600× speedup over CPU at N=2048. The optimized kernel matched or outperformed cuBLAS for smaller matrix sizes due to lower launch overhead. Packaged the CUDA kernel as a .so dynamic library callable from Python via ctypes — with convolution running 5.6× faster than pure C through the Python interface. Deployed and benchmarked on a Google Cloud GPU VM.

C CUDA C Shared Memory Tiling cuBLAS ctypes Google Cloud

HDFS Namespace Emulator with Persistent Snapshot

Node and Session classes support multi-user login, permission isolation, and a full shell command set (mkdir, touch, ls, cd, rm, rmdir).
